Visitor Management System — Architecture Spec
1. Overview & User Roles
This document specifies the end-to-end user flow, information architecture, navigation structure, and 
system architecture for an enterprise Visitor Management System (VMS). It is intended as a design and 
engineering handoff reference, exportable to Figma, FigJam, Lucidchart, Draw.io, and Miro.
User Roles
Role Description
Visitor External guest registering for and attending a visit
Receptionist Front-desk operator managing walk-ins, verification, and badges
Employee Internal host who approves/rejects visit requests
Security Guard Verifies QR passes, manages check-in/out, monitors live visitors
Admin Manages users, rooms, settings, and system-wide reports
Application Modules
•Authentication & Authorization
•Visitor Registration
•Employee Approval
•Reception Management
•QR Check-In / Check-Out
•Meeting Room Management
•Notifications
•Security Logs
•Reports & Analytics
•User & Role Management
•System Settings

7. Backend Architecture (Layered)
Requests flow through Express routes, controllers, middleware, and a service layer before reaching the 
Firestore/Storage data access layer. Logging, error handling, and caching are cross-cutting concerns 
applied at the controller and service layers.
Figure 10 — Layered backend architecture with cross-cutting concerns
Suggested Folder Structure
/src   /routes        -> visitor.routes.js, employee.routes.js, admin.routes.js   
/controllers   -> visitor.controller.js, approval.controller.js   /middleware    -> 
auth.middleware.js, rbac.middleware.js, validate.middleware.js   /services      -> 
qr.service.js, notification.service.js, report.service.js   /repositories  -> 
firestore.repository.js   /config        -> firebase.config.js, env.config.js   /utils  
-> logger.js, errorHandler.js   server.js

12. Design System Notes for Figma / FigJam Handoff
Color Coding
Blue = Visitor, Green = Employee, Orange = Receptionist, Red = Security, Purple = Admin, Yellow = 
Firebase / backend services.
Component Reuse
Build once and reuse across all five dashboards: Status Badge, QR Card, Dashboard Stat Card, 
Notification Toast, Approval Row.
Spacing & Grid
8px base spacing grid. 12-column responsive layout. Breakpoints at 1440 / 1024 / 768 / 375.
Typography Hierarchy
H1: 24-28px semibold (page titles)  ·  H2: 18-20px medium (section headers)  ·  Body: 14px regular  ·  
Caption: 12px (metadata/timestamps).
State Handling
Every flow node should have Figma variants for Default / Loading / Error / Empty states — especially 
Pending Approval, QR Scan, and Check-In steps.
